Sepomex API

Open REST API for Mexico's official postal-code (código postal) catalog — ~154,650 settlements across 32 states, ~2,475 municipalities and ~660 cities, from a bundled SQLite dataset. Read-only, public, no API key.

All endpoints live under the /api/v1 path. Prefix it with your deployment host — the examples below use http://localhost:3000 from a local run, but a hosted instance also runs at https://sepomex.kurenn.dev.

/api/v1
No API keyPublicCORS-enabledRead-only

Stack: Ruby 3.4 · Rails 8.1 (API-only) · SQLite · Puma.

Hosted instance: try it now at sepomex.kurenn.dev — a free, best-effort community instance (rate-limited, no SLA). Self-host for production; the dataset ships with the code.

AI agent? Skip the REST calls — query the same data as tools through the MCP server.

Health check
curl http://localhost:3000/up
First request
curl "http://localhost:3000/api/v1/zip_codes?zip_code=64000"

Conventions

All responses are JSON. Collections are wrapped in a resource root key (e.g. zip_codes) plus a meta object; a single resource uses its singular key (e.g. state).

List endpoints are paginated: 15 items per page by default, up to 200 via per_page (larger values fall back to 15). Combine with page. Every list response carries a meta.pagination block and the Link, X-Total-Pages and X-Total-Count response headers.

Interactive docs (Swagger UI) are served at /api-docs, and the raw OpenAPI 3 spec at /api-docs/v1/swagger.yaml.

meta.pagination
{
  "meta": {
    "pagination": {
      "per_page": 15,
      "total_pages": 9728,
      "total_objects": 145906,
      "links": {
        "first": "/api/v1/zip_codes?page=1",
        "last": "/api/v1/zip_codes?page=9728",
        "next": "/api/v1/zip_codes?page=2"
      }
    }
  }
}

Zip Codes

GET /api/v1/zip_codes

Search settlements (colonias) by any combination of the parameters below. All filters are partial and accent-insensitive.

ParamTypeDescription
zip_codestringPostal code, full or partial (e.g. "64000").
statestringState name (e.g. "nuevo leon").
citystringCity or municipality name (e.g. "monterrey").
colonystringColony / settlement name (e.g. "del valle").
per_pageintegerItems per page (default 15, max 200).
pageintegerPage number (default 1).
Requests
# by postal code
curl "http://localhost:3000/api/v1/zip_codes?zip_code=67173"

# by city / municipality
curl "http://localhost:3000/api/v1/zip_codes?city=monterrey"

# by state
curl "http://localhost:3000/api/v1/zip_codes?state=nuevo%20leon"

# by colony
curl "http://localhost:3000/api/v1/zip_codes?colony=del%20valle"

# combined
curl "http://localhost:3000/api/v1/zip_codes?state=nuevo%20leon&city=guadalupe"

States

GET /api/v1/states
GET /api/v1/states/:id
GET /api/v1/states/:id/municipalities

A state object is { id, name, cities_count }. The nested municipalities route returns that state's municipalities (unpaginated).

Requests
curl "http://localhost:3000/api/v1/states"
curl "http://localhost:3000/api/v1/states/1"
curl "http://localhost:3000/api/v1/states/1/municipalities"
Response
{ "state": { "id": 1, "name": "Ciudad de México", "cities_count": 16 } }

// GET /api/v1/states/1/municipalities
{
  "municipalities": [
    { "id": 1, "name": "Álvaro Obregón", "municipality_key": "010", "zip_code": "01001", "state_id": 1 }
  ]
}

Pagination

List endpoints accept page and per_page (default 15, max 200). Each response includes a meta.pagination block whose links preserve the current query string.

The same values are also returned as response headers:

HeaderDescription
Linkfirst / last / prev / next URLs (RFC 8288).
X-Total-PagesTotal number of pages.
X-Total-CountTotal number of records.
Request
curl -i "http://localhost:3000/api/v1/zip_codes?per_page=200&page=2"
Response headers
X-Total-Pages: 730
X-Total-Count: 145906
Link: <http://localhost:3000/api/v1/zip_codes?page=1&per_page=200>; rel="first"

Errors

Errors are returned as JSON with a consistent envelope: { error, message, status }.

StatusMeaning
200OK.
400Bad Request — a required parameter is missing.
404Not Found — no record with that :id.
429Too Many Requests — client is being rate-limited (includes Retry-After).
404 Not Found
{
  "error": "Not Found",
  "message": "Couldn't find State with 'id'=999999",
  "status": 404
}

MCP Server

Sepomex ships a Model Context Protocol server so AI agents (Claude Desktop, Claude Code, Cursor, …) can query the catalog as tools. Two transports serve the same tools:

POST /mcp — Streamable HTTP (stateless JSON-RPC)
bin/mcp — stdio, for local clients
ToolDescriptionArguments
lookup_zip_codeAll settlements for one exact CP.zip_code*
search_zip_codesFilter settlements.zip_code, state, city, colony, limit
list_statesThe 32 states.—
state_municipalitiesMunicipalities of a state.state_id*
search_citiesCities by name.query, limit

* required. Each tool returns a human-readable summary plus structuredContent (the same fields as the REST serializers).

Add it to a client

Claude Code (HTTP): claude mcp add --transport http sepomex http://localhost:3000/mcp

Local stdio: point your client's command at bin/mcp (see the config on the right), then restart the client.

HTTP — tools/call
curl -X POST http://localhost:3000/mcp \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call",
       "params":{"name":"lookup_zip_code",
                 "arguments":{"zip_code":"64000"}}}'
stdio — client config
{
  "mcpServers": {
    "sepomex": {
      "command": "/absolute/path/to/sepomex/bin/mcp"
    }
  }
}

Quickstart / Self-host

The dataset ships inside the app as SQLite — no external database to provision.

Refresh the dataset from the latest official SEPOMEX XML export with rake "data:import_xml[/path/to/CPdescarga.xml]", then rebuild the database.

Run it locally
git clone git@github.com:IcaliaLabs/sepomex.git
cd sepomex
bundle install
bin/rails db:prepare      # create SQLite DB + schema
bin/rake data:loadev      # import ~154k settlements
bin/rails server          # http://localhost:3000
